Software Development Task

GreenLife Organic Store is a rapidly growing retail company that specializes in organic groceries and eco-
friendly products. They aim to provide high-quality, sustainable products to their customers while maintaining
excellent service.
They require a desktop-based management system to handle their operations efficiently. The system should
cater to both Admin and Customer roles, providing a complete software solution for product management,
order processing, and reporting.
Functional Requirements

Admin Features
• Login – Secure login for admins.
• Manage Product Details – Add, update, delete product details (name, category, price, stock, supplier).
• Manage Customer Details – View and update customer information.
• Manage Orders – View all orders, update order status (Pending, Shipped, Delivered).
• Generate Reports – Sales report, stock report, and customer order history.
• Dashboard – Overview of total sales, products in stock, and active orders.

Customer Features
• Register/Login – New user registration and login.
• Search Products – Search by name, category, or price range.
• Place Orders – Add products to cart and place orders.
• Track Orders – View the status of placed orders.
• Profile Management – Update personal details.

Additional Features
• Apply discounts on products.
• Include product ratings and reviews.
• Low-stock notifications for Admin.
• Order history export (CSV or PDF).

4

Technical Requirements

• Programming Language: C# (.NET Framework, Windows Forms Application)
• Data Storage: Local database (SQL Server or XML/JSON files for simplicity).
• Software Design:
o Use classes to represent core entities: Product, Customer, Order, Admin, etc.
o Implement appropriate properties and methods.
o Include inheritance or interfaces where suitable.
• User Interface:
o Window-based forms for all functionalities.
o Separate forms for Admin and Customer tasks.
o Navigation menu/dashboard for easy access.
• Validation &amp; Exception Handling:
o Validate user inputs (e.g., numeric fields, empty inputs).
o Handle exceptions to prevent application crashes.
• Search Functionality:
o Implement search algorithms (e.g., linear search, filtering, or sorting) for products.
o Optional: implement more efficient search methods if desired (e.g., binary search for sorted lists).

Note: Additionally, you may add extra features (data, functionality, and technical) to the application, if you
wish.
